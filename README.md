# 部署與資源說明

- 前端、後端皆部署於 Render 雲端平台
- 資料庫使用 Railway 雲端 MySQL
- 商品圖片資源存放於 Cloudinary 雲端圖片服務

## 資料庫設計

### 資料表結構
1. **商品表（`products`）**
   - 欄位：id, name, price, image, stock, description
2. **訂單表（`orders`）**
   - 欄位：id, user_id, product_list, total_amount, status, tradeNo, note
### 商品表（products）
- id：主鍵，自動編號
- name：商品名稱
- price：商品價格
- image：商品圖片網址
- stock：庫存數量
- description：商品描述

### 訂單表（orders）
- id：主鍵，自動編號
- user_id：用戶編號
- product_list：商品清單
- total_amount：總金額
- status：訂單狀態
- tradeNo：金流交易編號
- note：訂單備註
- address:地址
# 要實作購物車系統

## 程式語言的選擇
- 前端使用 React.js
- 後端使用 Node.js
- 串接 MySQL

## 功能需求

### 前台功能
1. **商品瀏覽**
   - 用戶可以瀏覽商品列表。
2. **購物車管理**
   - 用戶可以將商品加入購物車。
   - 購物車頁面顯示已加入的商品清單和總金額。
3. **結帳流程**
   - 點擊購物車圖案後跳轉到購物車頁面。
   - 確認商品後跳轉到金流頁面（綠界 API）。
   - 完成結帳後顯示訂單完成頁面。

### 後台功能
1. **商品管理**
   - 商店用戶可以新增、修改、刪除商品。
2. **訂單管理**
   - 商店用戶可以查看被下的訂單。
   - 訂單詳情包括商品清單、總金額、訂單狀態。



## 技術細節

### 前端
- 使用 React.js 開發。
- 使用狀態管理工具（如 Redux 或 Context API）管理購物車狀態。

### 後端
- 使用 Node.js 框架（如 Express.js）處理 API 請求。
- 使用 MySQL 儲存資料，並設計資料表結構。

### 金流
- 使用綠界 API 提供的付款功能（如信用卡付款、ATM 轉帳）。

## 非功能性需求
- **性能要求**：系統需在 1 秒內響應用戶操作。
- **安全性**：
  - 金流頁面需使用 HTTPS。
  - 防止 SQL 注入攻擊。

## 流程圖

### 用戶操作流程
1. 用戶瀏覽商品列表。
2. 選擇商品並加入購物車。
3. 點擊購物車圖案，跳轉到購物車頁面。
4. 確認商品後跳轉到金流頁面。
5. 完成結帳，顯示訂單完成頁面。

### 商店用戶操作流程
1. 登入後進入後台。
2. 新增、修改或刪除商品。
3. 查看被下的訂單。
